Datatypes

Objects and Methods

Defining an Object

var someObject = new Object()
var otherObject = {}

Adding properties and Values
  • someObject.propertyName = value (dot notation)
  • someObject["propertyName"] = value (bracket notation)
Accessing properties
  • someObject.propertyName
  • someObject["propertyName"]
Accessing nested properties
  • someObject.propertyName1.propertyName1.1
  • someObject["propertyName1"]["propertyName1.1"]
  • someObject.propertyName1["propertyName1.1"]
Deleting properties
delete someObject.propertyName
Object.keys(someObject)
"returns an array of all properties"
Object.keys(someObject.propertyName)
"returns an array of all properties but for nested objects"

Conditional

if statement

if (condition) {do something}
else if (condition) {do something}
else {do something}
if you have a simple one line code for each conditional, than ot's not obligatory to use the {}; however, if you have a code of more than one statement, than it's better to use the {}, otherwise your code will be executed not just for one condition but for more than one.

switch

switch (var name){
case 1:
do something
break;
case 2:
do something
break;
default:
do something
break;
}
never forget the break, otherwise you will have case 1 and case 2 executed at the same time.
